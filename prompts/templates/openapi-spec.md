## 生成してほしい内容

完全なOpenAPI 3.0仕様書（YAML形式）を生成してください。以下を含めること：

1. **openapi, info, serversセクション**
2. **pathsセクション** - すべてのエンドポイント定義
3. **componentsセクション**:
   - schemas: データモデル定義
   - securitySchemes: 認証方式定義
   - responses: 共通レスポンス定義
   - parameters: 共通パラメータ定義
4. **tagsセクション** - エンドポイントの分類

各エンドポイントには以下を含めてください：
- summary, description
- parameters (path, query, header)
- requestBody (該当する場合)
- responses (各ステータスコード)
- security要件

すべてのスキーマには適切なバリデーション（required, format, pattern, minLength, maxLengthなど）を含めてください。

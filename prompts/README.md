# プロンプト集

このディレクトリには、AIエージェントを使用してAPI仕様書を生成するためのプロンプトテンプレートが含まれています。

## ディレクトリ構造

```
prompts/
├── templates/          # 再利用可能なプロンプトテンプレート
│   ├── basic-api-spec.md
│   ├── openapi-spec.md
│   ├── endpoint-detail.md
│   └── error-handling.md
├── examples/           # 実際の使用例
│   └── user-management-api.md
└── README.md          # このファイル
```

## テンプレート一覧

### 1. basic-api-spec.md
基本的なAPI仕様書を生成するためのテンプレート。
RESTful APIの概要、エンドポイント、認証方式などを含む標準的な仕様書を生成します。

**対象者**: 初めてAPI仕様書を作成する人、小規模APIのドキュメントを作りたい人

### 2. openapi-spec.md
OpenAPI 3.0形式の仕様書（YAML）を生成するためのテンプレート。
Swagger UIなどのツールで使用できる標準形式の仕様書を生成します。

**対象者**: OpenAPI標準に準拠した仕様書が必要な人、ツール連携を考えている人

### 3. endpoint-detail.md
特定のエンドポイントについて詳細な仕様書を生成するためのテンプレート。
リクエスト、レスポンス、エラー処理、使用例などを網羅的に記載します。

**対象者**: 複雑なエンドポイントの詳細ドキュメントが必要な人

### 4. error-handling.md
API全体のエラーハンドリング戦略を定義するためのテンプレート。
エラーコード、エラーメッセージ形式、ベストプラクティスなどを定義します。

**対象者**: エラー処理を標準化したい人、チーム全体で統一したい人

## 使い方

### 基本的な流れ

1. **テンプレートの選択**: プロジェクトに適したテンプレートを選ぶ
2. **カスタマイズ**: `[...]` の部分を実際の情報で置き換える
3. **AI入力**: カスタマイズしたプロンプトをAIエージェントに入力
4. **生成**: AIが仕様書を自動生成
5. **確認・調整**: 生成された仕様書を確認し、必要に応じて修正

### 例: 基本的なAPI仕様書の作成

```bash
# 1. テンプレートをコピー
cp prompts/templates/basic-api-spec.md my-prompt.md

# 2. エディタで編集して [API名を記入] などを実際の値に置き換え

# 3. AIエージェント（ChatGPT, Claude, Copilotなど）に貼り付け

# 4. 生成された仕様書を保存
```

## 例（examples/）

`examples/` ディレクトリには、テンプレートを実際に使用した例が含まれています。
これらを参考にして、自分のプロジェクト用のプロンプトを作成できます。

### user-management-api.md
ユーザー管理APIの仕様書を生成する完全な例。
基本的なCRUD操作を含むRESTful APIのプロンプト例です。

## カスタマイズのヒント

### 1. 具体的な情報を含める
AIエージェントは具体的な情報があるほど良い仕様書を生成します。

❌ 悪い例: `API名: [API名]`
✅ 良い例: `API名: E-Commerce Product API v2.0`

### 2. データ例を提供する
リクエスト/レスポンスの具体例を含めると、より実用的な仕様書になります。

### 3. 段階的に詳細化する
最初は全体像（basic-api-spec）から始めて、徐々に詳細化（endpoint-detail）していくのが効果的です。
